# Настройка тестовой базы данных

## ✅ Реализовано

Создана **отдельная тестовая инфраструктура** с полной изоляцией от продакшн данных.

### Архитектура

```
┌─────────────────────────────────────┐
│  Production Environment              │
│  ┌─────────────────────────────┐     │
│  │ PostgreSQL (port 5432)      │     │
│  │ Database: randomcoffee      │     │
│  └─────────────────────────────┘     │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│  Test Environment                    │
│  ┌─────────────────────────────┐     │
│  │ PostgreSQL (port 5434)      │     │
│  │ Database: randomcoffee_test │     │
│  │ Storage: tmpfs (in-memory)  │     │
│  └─────────────────────────────┘     │
└─────────────────────────────────────┘
```

### Компоненты

1. **`docker-compose.test.yml`** - отдельный compose файл для тестов
   - Контейнер: `randomcoffee-db-test`
   - Порт: `5434` (не конфликтует с продакшн на 5432)
   - Хранилище: `tmpfs` (in-memory, быстрее)
   - Автоматическая очистка при остановке

2. **`tests/conftest.py`** - обновленная конфигурация
   - Подключение к порту 5434 по умолчанию
   - Автоматическое создание тестовой БД
   - Транзакционная изоляция для каждого теста

3. **Makefile** - удобные команды
   - `make test` - автоматически запускает/останавливает БД
   - `make test-db-up` - запустить тестовую БД
   - `make test-db-down` - остановить тестовую БД
   - `make test-coverage` - тесты с покрытием

## 🚀 Использование

### Быстрый старт

```bash
# Запустить все тесты (автоматически управляет БД)
make test

# Или вручную:
make test-db-up    # Запустить тестовую БД
uv run pytest tests/ -v
make test-db-down  # Остановить тестовую БД
```

### Преимущества

1. ✅ **Полная изоляция:** Отдельный контейнер, отдельный порт, отдельная БД
2. ✅ **Быстрота:** tmpfs для in-memory хранилища
3. ✅ **Безопасность:** Продакшн БД никогда не используется
4. ✅ **Автоматизация:** Makefile команды для удобства
5. ✅ **Параллельность:** Можно запускать тесты параллельно с продакшн

## 📋 Best Practices

### В промышленной разработке обычно используют:

1. **Отдельный docker-compose для тестов** ✅ (реализовано)
   - Изоляция на уровне контейнеров
   - Легко очищать (удалить контейнер)

2. **Testcontainers** (альтернатива)
   - Автоматическое создание контейнеров в тестах
   - Хорошо для CI/CD

3. **Транзакционная изоляция** ✅ (реализовано)
   - Каждый тест в отдельной транзакции
   - Автоматический rollback

4. **In-memory хранилище для тестов** ✅ (реализовано)
   - tmpfs для скорости
   - Автоматическая очистка

## 🔧 Конфигурация

### Переменные окружения

```bash
# Настройка тестовой БД
export TEST_DATABASE_NAME="randomcoffee_test"
export TEST_DATABASE_HOST="localhost"
export TEST_DATABASE_PORT="5434"  # Отдельно от продакшн (5432)
export TEST_DATABASE_USER="postgres"
export TEST_DATABASE_PASSWORD="postgres"

# Или полный URL
export TEST_DATABASE_URL="postgresql+asyncpg://user:pass@host:port/db"
```

### Docker Compose

```yaml
# docker-compose.test.yml
services:
  db-test:
    image: postgres:16-alpine
    ports:
      - "5434:5432"  # Отдельный порт
    tmpfs:
      - /var/lib/postgresql/data  # In-memory
```

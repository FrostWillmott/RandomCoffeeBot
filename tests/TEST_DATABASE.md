# Конфигурация тестовой базы данных

## Текущая конфигурация

Тесты используют **отдельный контейнер PostgreSQL** для полной изоляции от продакшн данных.

### Используемое решение

**PostgreSQL через Docker Compose** (`docker-compose.test.yml`):
- Отдельный контейнер: `randomcoffee-db-test`
- Отдельный порт: `5434` (продакшн использует `5432`)
- Отдельная БД: `randomcoffee_test`
- In-memory хранилище: используется `tmpfs` для скорости
- Автоматическая очистка: контейнер не сохраняет данные после остановки

### Преимущества

1. **Полная совместимость** с продакшн БД (PostgreSQL 16)
2. **Изоляция** от продакшн данных
3. **Быстрота** благодаря tmpfs (in-memory хранилище)
4. **Автоматизация** через Makefile команды
5. **Безопасность** - продакшн БД никогда не используется

## Настройка

### Автоматическое управление (рекомендуется)

```bash
# Запустить тесты (автоматически запустит/остановит БД)
make test
```

### Ручное управление

```bash
# Запустить тестовую БД
make test-db-up
# или
docker-compose -f docker-compose.test.yml up -d db-test

# Запустить тесты
uv run pytest tests/ -v

# Остановить тестовую БД
make test-db-down
# или
docker-compose -f docker-compose.test.yml down
```

## Переменные окружения

В `tests/conftest.py` настраиваются следующие переменные:

- `TEST_DATABASE_NAME` - имя тестовой БД (по умолчанию: `randomcoffee_test`)
- `TEST_DATABASE_HOST` - хост БД (по умолчанию: `localhost`)
- `TEST_DATABASE_PORT` - порт БД (по умолчанию: `5434`)
- `TEST_DATABASE_USER` - пользователь БД (по умолчанию: `postgres`)
- `TEST_DATABASE_PASSWORD` - пароль БД (по умолчанию: `postgres`)
- `TEST_DATABASE_URL` - полный URL тестовой БД (переопределяет все выше)

## Изоляция тестов

Каждый тест запускается в отдельной транзакции, которая автоматически откатывается после завершения. Это обеспечивает:
- Полную изоляцию между тестами
- Автоматическую очистку данных
- Быстроту выполнения (транзакции быстрее полной очистки БД)

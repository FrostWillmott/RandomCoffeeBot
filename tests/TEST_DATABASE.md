# Конфигурация тестовой базы данных

## Текущая ситуация

Тесты используют **SQLite in-memory** базу данных по умолчанию, но модель `Topic` использует PostgreSQL ARRAY, который SQLite не поддерживает.

## Варианты решения

### Вариант 1: Использовать PostgreSQL для тестов (рекомендуется)
**Плюсы:**
- Полная совместимость с продакшн БД
- Тестирование на реальной БД
- Поддержка всех функций PostgreSQL

**Минусы:**
- Требует запущенной PostgreSQL БД
- Медленнее чем SQLite

**Настройка:**
```bash
# Запустить PostgreSQL через docker-compose
docker-compose up -d db

# Или установить переменную окружения
export TEST_DATABASE_URL="postgresql+asyncpg://postgres:postgres@localhost:5432/randomcoffee_test"
```

### Вариант 2: Адаптировать модель для SQLite
**Плюсы:**
- Быстрые тесты без внешней БД
- Не требует запущенной БД

**Минусы:**
- Требует изменения модели
- Не тестирует реальное поведение PostgreSQL

**Изменения:**
- Заменить `ARRAY(Text)` на `JSON` для SQLite
- Использовать условную логику в модели

### Вариант 3: Использовать отдельную тестовую БД
**Плюсы:**
- Изоляция от продакшн данных
- Полная совместимость

**Минусы:**
- Требует настройки отдельной БД

## Рекомендация

**Использовать PostgreSQL для тестов** (Вариант 1), так как:
1. Проект использует PostgreSQL в продакшене
2. Тесты должны проверять реальное поведение
3. Docker-compose уже настроен

## Текущая конфигурация

В `tests/conftest.py`:
- По умолчанию: SQLite in-memory (`sqlite+aiosqlite:///:memory:`)
- Можно переопределить через `TEST_DATABASE_URL`

## Быстрый старт

```bash
# Запустить PostgreSQL
docker-compose up -d db

# Установить тестовую БД URL
export TEST_DATABASE_URL="postgresql+asyncpg://postgres:postgres@localhost:5432/randomcoffee_test"

# Запустить тесты
uv run pytest tests/ -v
```
